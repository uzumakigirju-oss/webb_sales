# 🔍 Отчет об ошибках и проблемах проекта Omnom & SweetMe

## 🔴 КРИТИЧЕСКИЕ ОШИБКИ

### 1. **Missing Dependency: `python-multipart`**
- **Статус**: ✅ ИСПРАВЛЕНО
- **Файл**: `web_app.py` (линия 20)
- **Ошибка**: 
  ```
  RuntimeError: Form data requires "python-multipart" to be installed.
  ```
- **Решение**: Установлен `python-multipart==0.0.6` в `requirements.txt`

---

### 2. **Компрометированный BOT_TOKEN** 🔐
- **Статус**: ⚠️ ТРЕБУЕТ ДЕЙСТВИЯ
- **Файл**: `.env`
- **Проблема**: Токен виден в версионной системе и может быть украден
- **Действия**:
  1. Отправьте `/start` боту **BotFather** в Telegram
  2. Выберите ваш бот
  3. Нажмите "Edit Bot" → "Edit Tokens"
  4. Сгенерируйте новый токен
  5. Обновите `.env`
  6. Добавьте `.env` в `.gitignore`

---

## ⚠️ ВАЖНЫЕ ПРЕДУПРЕЖДЕНИЯ

### 3. **CORS не настроен**
- **Статус**: ✅ ИСПРАВЛЕНО
- **Файл**: `web_app.py`
- **Проблема**: Веб-приложение может не работать при кросс-доменных запросах
- **Решение**: Добавлена `CORSMiddleware` в приложение

---

### 4. **Хрупкая обработка CSV формата**
- **Статус**: ✅ УЛУЧШЕНО
- **Файл**: `sales_manager.py` (строки 41-52)
- **Проблема**: 
  ```python
  price = int(row[1].strip())  # Падает на " 160" (строка с пробелом)
  ```
  В `products.csv` есть строки вида `' 160'` (с пробелом)
- **Решение**: Улучшена обработка:
  ```python
  price = int(float(row[1].strip()))  # Теперь работает с пробелами и float
  ```

---

## 🟡 ПОТЕНЦИАЛЬНЫЕ ПРОБЛЕМЫ

### 5. **Нет валидации ввода в чеках**
- **Файл**: `handlers/sales.py` и `web_app.py:368-398`
- **Проблема**: 
  ```python
  qty = int(item.get("qty", 1))  # Может быть отрицательным!
  price = int(item["price"])      # Не проверяется на корректность
  ```
- **Рекомендация**: Добавить валидацию:
  ```python
  if qty <= 0:
      raise ValueError("Количество должно быть > 0")
  if price < 0:
      raise ValueError("Цена не может быть отрицательной")
  ```

---

### 6. **Отсутствует обработка ошибок БД**
- **Файл**: `database.py`
- **Проблема**: Нет try-except при обращении к БД
- **Рекомендация**: Добавить обработку:
  ```python
  def db_get_user_fair(user_id: int) -> Optional[str]:
      try:
          with sqlite3.connect(DB_PATH) as conn:
              row = conn.execute(...)
      except sqlite3.OperationalError as e:
          logger.error(f"DB error: {e}")
          return None
  ```

---

### 7. **Гонка данных в многопроцессной среде**
- **Файл**: `main.py:11-27` (kill_other_instances)
- **Проблема**: Может быть race condition между проверкой PID и его убийством
- **Рекомендация**: Использовать файловый lock вместо сигналов

---

### 8. **Кэш товаров может рассинхронизироваться**
- **Файл**: `sales_manager.py:22-25`
- **Проблема**: 
  ```python
  _products_cache = {"data": None, "mtime": 0.0}
  # Проверяется только mtime, но не содержимое
  ```
- **Риск**: Если mtime не меняется (быстрое редактирование), кэш будет старый

---

### 9. **Нет логирования критических операций**
- **Файл**: `web_app.py` (функции API)
- **Проблема**: Недостаточно информации для отладки в production
- **Рекомендация**: Добавить логирование всех API запросов
